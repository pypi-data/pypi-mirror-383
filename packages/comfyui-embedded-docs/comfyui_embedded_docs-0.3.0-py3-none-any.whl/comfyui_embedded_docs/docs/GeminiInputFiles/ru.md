Загружает и форматирует входные файлы для использования с API Gemini. Этот узел позволяет пользователям включать текстовые (.txt) и PDF (.pdf) файлы в качестве входного контекста для модели Gemini. Файлы преобразуются в соответствующий формат, требуемый API, и могут быть объединены в цепочку для включения нескольких файлов в один запрос.

## Входы

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `file` | COMBO | Да | Доступно несколько вариантов | Входные файлы для включения в контекст модели. Пока принимаются только текстовые (.txt) и PDF (.pdf) файлы. Файлы должны быть меньше максимального ограничения на размер входного файла. |
| `GEMINI_INPUT_FILES` | GEMINI_INPUT_FILES | Нет | N/A | Дополнительный файл(ы) для пакетной обработки вместе с файлом, загруженным из этого узла. Позволяет объединять входные файлы в цепочку, чтобы одно сообщение могло включать несколько входных файлов. |

**Примечание:** Параметр `file` отображает только текстовые (.txt) и PDF (.pdf) файлы, которые меньше максимального ограничения на размер входного файла. Файлы автоматически фильтруются и сортируются по имени.

## Выходы

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `GEMINI_INPUT_FILES` | GEMINI_INPUT_FILES | Отформатированные данные файла, готовые к использованию с узлами LLM Gemini, содержащие загруженное содержимое файла в соответствующем формате для API. |
