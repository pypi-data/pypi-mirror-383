Этот узел обнаруживает модели, расположенные в папке `ComfyUI/models/checkpoints`, а также читает модели из дополнительных путей, настроенных в файле extra_model_paths.yaml. Иногда может потребоваться **обновить интерфейс ComfyUI**, чтобы он смог прочитать файлы моделей из соответствующей папки.

Узел unCLIPCheckpointLoader предназначен для загрузки контрольных точек, специально адаптированных для моделей unCLIP. Он обеспечивает извлечение и инициализацию моделей, модулей CLIP vision и VAEs из указанной контрольной точки, упрощая процесс настройки для последующих операций или анализов.

## Входы

| Поле        | Comfy тип     | Описание                                                                           |
|-------------|---------------|-----------------------------------------------------------------------------------|
| `название_ckpt` | `COMBO[STRING]`| Указывает имя загружаемой контрольной точки, идентифицируя и извлекая правильный файл контрольной точки из предопределённого каталога, определяя инициализацию моделей и конфигураций. |

## Выходы

| Поле         | Comfy тип    | Описание                                                              | Python тип          |
|--------------|--------------|----------------------------------------------------------------------|---------------------|
| `model`      | `MODEL`      | Представляет основную модель, загруженную из контрольной точки.        | `torch.nn.Module`   |
| `clip`       | `CLIP`       | Представляет модуль CLIP, загруженный из контрольной точки, если он доступен. | `torch.nn.Module`   |
| `vae`        | `VAE`        | Представляет модуль VAE, загруженный из контрольной точки, если он доступен.  | `torch.nn.Module`   |
| `clip_vision`| `CLIP_VISION`| Представляет модуль CLIP vision, загруженный из контрольной точки, если он доступен. | `torch.nn.Module`   |
