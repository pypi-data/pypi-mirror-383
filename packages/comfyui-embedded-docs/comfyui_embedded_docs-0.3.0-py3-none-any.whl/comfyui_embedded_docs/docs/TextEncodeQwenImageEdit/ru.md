Узел TextEncodeQwenImageEdit обрабатывает текстовые промпты и опциональные изображения для генерации кондиционирующих данных, используемых при создании или редактировании изображений. Он использует модель CLIP для токенизации входных данных и может дополнительно кодировать эталонные изображения с помощью VAE для создания референсных латентов. При предоставлении изображения узел автоматически изменяет его размер для поддержания согласованных размеров обработки.

## Входы

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `clip` | CLIP | Да | - | Модель CLIP, используемая для токенизации текста и изображений |
| `prompt` | STRING | Да | - | Текстовый промпт для генерации кондиционирования, поддерживает многострочный ввод и динамические промпты |
| `vae` | VAE | Нет | - | Опциональная модель VAE для кодирования эталонных изображений в латенты |
| `image` | IMAGE | Нет | - | Опциональное входное изображение для использования в качестве референса или для редактирования |

**Примечание:** Когда предоставлены и `image`, и `vae`, узел кодирует изображение в референсные латенты и присоединяет их к выходным кондиционирующим данным. Изображение автоматически масштабируется для поддержания согласованного масштаба обработки приблизительно 1024x1024 пикселей.

## Выходы

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `CONDITIONING` | CONDITIONING | Кондиционирующие данные, содержащие текстовые токены и опциональные референсные латенты для генерации изображений |
