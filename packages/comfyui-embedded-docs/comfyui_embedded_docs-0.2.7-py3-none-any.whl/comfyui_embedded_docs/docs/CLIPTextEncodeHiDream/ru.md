Узел CLIPTextEncodeHiDream обрабатывает несколько текстовых входов с использованием различных языковых моделей и объединяет их в единый выход кондиционирования. Он токенизирует текст из четырех различных источников (CLIP-L, CLIP-G, T5-XXL и LLaMA) и кодирует их с использованием подхода планового кодирования. Это позволяет осуществлять более сложное текстовое кондиционирование за счет одновременного использования нескольких языковых моделей.

## Входы

| Параметр | Тип данных | Тип ввода | По умолчанию | Диапазон | Описание |
|-----------|-----------|------------|---------|-------|-------------|
| `clip` | CLIP | Обязательный вход | - | - | Модель CLIP, используемая для токенизации и кодирования |
| `clip_l` | STRING | Многострочный текст | - | - | Текстовый вход для обработки моделью CLIP-L |
| `clip_g` | STRING | Многострочный текст | - | - | Текстовый вход для обработки моделью CLIP-G |
| `t5xxl` | STRING | Многострочный текст | - | - | Текстовый вход для обработки моделью T5-XXL |
| `llama` | STRING | Многострочный текст | - | - | Текстовый вход для обработки моделью LLaMA |

**Примечание:** Все текстовые входы поддерживают динамические промпты и многострочный ввод текста. Для корректной работы узла необходимо предоставить все четыре текстовых параметра, так как каждый из них вносит вклад в итоговый выход кондиционирования через процесс планового кодирования.

## Выходы

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `CONDITIONING` | CONDITIONING | Объединенный выход кондиционирования от всех обработанных текстовых входов |
