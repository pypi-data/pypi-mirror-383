Узел CheckpointLoader загружает предварительно обученную модель из контрольной точки вместе с её конфигурационным файлом. Он принимает конфигурационный файл и файл контрольной точки в качестве входных данных и возвращает загруженные компоненты модели, включая основную модель, модель CLIP и модель VAE для использования в рабочем процессе.

## Входные параметры

| Параметр | Тип данных | Тип ввода | По умолчанию | Диапазон | Описание |
|-----------|-----------|------------|---------|-------|-------------|
| `название_конфигурации` | STRING | COMBO | - | Доступные конфигурационные файлы | Конфигурационный файл, определяющий архитектуру и настройки модели |
| `название_точки_проверки` | STRING | COMBO | - | Доступные файлы контрольных точек | Файл контрольной точки, содержащий обученные веса и параметры модели |

**Примечание:** Для работы этого узла необходимо выбрать как конфигурационный файл, так и файл контрольной точки. Конфигурационный файл должен соответствовать архитектуре загружаемого файла контрольной точки.

## Выходные данные

| Выходной параметр | Тип данных | Описание |
|-------------|-----------|-------------|
| `MODEL` | MODEL | Загруженный основной компонент модели, готовый к использованию для вывода |
| `CLIP` | CLIP | Загруженный компонент модели CLIP для текстового кодирования |
| `VAE` | VAE | Загруженный компонент модели VAE для кодирования и декодирования изображений |

**Важное примечание:** Этот узел помечен как устаревший и может быть удалён в будущих версиях. Рекомендуется использовать альтернативные узлы загрузки для новых рабочих процессов.
