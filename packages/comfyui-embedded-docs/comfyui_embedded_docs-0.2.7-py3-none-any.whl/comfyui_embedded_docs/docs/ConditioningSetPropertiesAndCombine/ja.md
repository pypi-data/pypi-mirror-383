ConditioningSetPropertiesAndCombineノードは、新しいコンディショニング入力から既存のコンディショニング入力にプロパティを適用することで、コンディショニングデータを変更します。2つのコンディショニングセットを結合しながら、新しいコンディショニングの強度を制御し、コンディショニング領域の適用方法を指定します。

## 入力

| パラメータ | データ型 | 入力タイプ | デフォルト | 範囲 | 説明 |
|-----------|-----------|------------|---------|-------|-------------|
| `条件` | CONDITIONING | 必須 | - | - | 変更対象の元のコンディショニングデータ |
| `新しい条件` | CONDITIONING | 必須 | - | - | 適用するプロパティを提供する新しいコンディショニングデータ |
| `強度` | FLOAT | 必須 | 1.0 | 0.0 - 10.0 | 新しいコンディショニングプロパティの強度を制御します |
| `条件付けエリア設定` | STRING | 必須 | default | ["default", "mask bounds"] | コンディショニング領域の適用方法を決定します |
| `マスク` | MASK | オプション | - | - | コンディショニングを適用する特定の領域を定義するオプションのマスク |
| `フック` | HOOKS | オプション | - | - | カスタム処理のためのオプションのフック関数 |
| `タイムステップ` | TIMESTEPS_RANGE | オプション | - | - | コンディショニングを適用するタイミングを制御するオプションのタイムステップ範囲 |

**注記:** `mask`が提供された場合、`set_cond_area`パラメータで"mask bounds"を使用して、コンディショニングの適用をマスクされた領域に制限できます。

## 出力

| 出力名 | データ型 | 説明 |
|-------------|-----------|-------------|
| `CONDITIONING` | CONDITIONING | 変更されたプロパティを持つ結合されたコンディショニングデータ |
