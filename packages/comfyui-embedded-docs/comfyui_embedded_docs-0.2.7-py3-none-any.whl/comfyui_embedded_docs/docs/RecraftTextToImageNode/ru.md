Генерирует изображения синхронно на основе промпта и разрешения. Этот узел подключается к API Recraft для создания изображений из текстовых описаний с указанными размерами и параметрами стиля.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `prompt` | STRING | Да | - | Промпт для генерации изображения. (по умолчанию: "") |
| `size` | COMBO | Да | "1024x1024"<br>"1152x896"<br>"896x1152"<br>"1216x832"<br>"832x1216"<br>"1344x768"<br>"768x1344"<br>"1536x640"<br>"640x1536" | Размер генерируемого изображения. (по умолчанию: "1024x1024") |
| `n` | INT | Да | 1-6 | Количество генерируемых изображений. (по умолчанию: 1) |
| `seed` | INT | Да | 0-18446744073709551615 | Сид для определения, должен ли узел повторно выполняться; фактические результаты недетерминированы независимо от сида. (по умолчанию: 0) |
| `recraft_style` | COMBO | Нет | Доступно несколько опций | Опциональный выбор стиля для генерации изображения. |
| `negative_prompt` | STRING | Нет | - | Опциональное текстовое описание нежелательных элементов на изображении. (по умолчанию: "") |
| `recraft_controls` | COMBO | Нет | Доступно несколько опций | Опциональные дополнительные элементы управления генерацией через узел Recraft Controls. |

**Примечание:** Параметр `seed` управляет только тем, когда узел повторно выполняется, но не делает генерацию изображения детерминированной. Фактические выходные изображения будут различаться даже при одинаковом значении сида.

## Выходные данные

| Выходной параметр | Тип данных | Описание |
|-------------|-----------|-------------|
| `IMAGE` | IMAGE | Сгенерированное(ые) изображение(я) в виде тензорного вывода. |
