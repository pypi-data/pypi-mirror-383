## üìñ Introduction

KumoRFM is a pre-trained *Relational Foundation Model (RFM)* that generates training-free predictions on any relational multi-table data by interpreting the data as a (temporal) heterogeneous graph.
It can be queried via the *Predictive Query Language (PQL)*.

This repository hosts a full-featured *MCP (Model Context Protocol)* server that empowers AI assistants with KumoRFM intelligence.
This server enables:

- üï∏Ô∏è Build, manage, and visualize graphs directly from CSV or Parquet files
- üí¨ Convert natural language into PQL queries for seamless interaction
- ü§ñ Query, analyze, and evaluate predictions from KumoRFM (missing value imputation, temporal forecasting, *etc*) all without any training required

## üöÄ Installation

### üêç Traditional MCP Server

The KumoRFM MCP server is available for Python 3.10 and above. To install, simply run:

```bash
pip install kumo-rfm-mcp
```

Add to your MCP configuration file (*e.g.*, Claude Desktop's `mcp_config.json`):

```json
{
  "mcpServers": {
    "kumo-rfm": {
      "command": "python",
      "args": ["-m", "kumo_rfm_mcp.server"],
      "env": {
        "KUMO_API_KEY": "your_api_key_here"
      }
    }
  }
}
```

### ‚ö° MCP Bundle

We provide a single-click installation via our [MCP Bundle (MCPB)](https://github.com/anthropics/mcpb) (*e.g.*, for integration into Claude Desktop):

1. Download the `dxt` file from [here](https://kumo-sdk-public.s3.us-west-2.amazonaws.com/dxt/kumo-rfm-mcp-0.1.0.dxt)
1. Double click to install

The MCP Bundle supports Linux, macOS and Windows, but requires a Python executable to be found in order to create a separate new virtual environment.

## üìö Available Tools

### I/O Operations

- **üîç `find_table_files` - Searching for tabular files:** Find all table-like files (*e.g.*, CSV, Parquet) in a directory.
- **üßê `inspect_table_files` - Analyzing table structure:** Inspect the first rows of table-like files.

### Graph Management

- **üóÇÔ∏è `inspect_graph_metadata` - Reviewing graph schema:** Inspect the current graph metadata.
- **üîÑ `update_graph_metadata` - Updating graph schema:** Partially update the current graph metadata.
- **üñºÔ∏è `get_mermaid` - Creating graph diagram:** Return the graph as a Mermaid entity relationship diagram.
- **üï∏Ô∏è `materialize_graph` - Assembling graph:** Materialize the graph based on the current state of the graph metadata to make it available for inference operations.
- **üìÇ `lookup_table_rows` - Retrieving table entries:** Lookup rows in the raw data frame of a table for a list of primary keys.

### Model Execution

- **ü§ñ `predict` - Running predictive query:** Execute a predictive query and return model predictions.
- **üìä `evaluate` - Evaluating predictive query:** Evaluate a predictive query and return performance metrics which compares predictions against known ground-truth labels from historical examples.
- **üß† `explain` - Explaining prediction:** Execute a predictive query and explain the model prediction.

## üîß Configuration

### Environment Variables

- **`KUMO_API_KEY`:** Authentication is needed once before predicting or evaluating with the
  KumoRFM model.
  You can generate your KumoRFM API key for free [here](https://kumorfm.ai).
  If not set, you can also authenticate on-the-fly in individual session via an OAuth2 flow.
